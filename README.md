# 流年的小手机开发教程

> AI-Native 跨平台应用开发教程

这是一个面向新手的小手机开发教程，帮助你从零开始构建属于自己的 AI 陪伴应用。

## 什么是"小手机"？

小手机是一种基于浏览器的 AI-Native 产品，通过 HTML + JavaScript + PWA 技术栈，在浏览器中实现一个虚拟的操作系统。你可以在里面创建微信、QQ、抖音等常见应用的虚拟版本，并通过自定义 AI 人设，打造完美真实的 AI Roleplay 闭环，提升陪伴体验。

## 核心特性

- **🚀 AI 优先** - 所有交互逻辑、数据处理均围绕 AI 能力构建，无缝衔接自定义大模型
- **⚡ 极致工程化** - 采用 HTML + JavaScript 构建跨平台应用，统一管理构建、测试与部署流程
- **📱 原生体验** - 通过 PWA 技术实现接近原生的用户体验，一套代码覆盖多个平台

## 教程内容

本教程包含以下内容：

- **准备工作** - 开发工具、AI 模型选择、逻辑化表达能力培养
- **环境搭建** - 从零开始配置开发环境
- **快速入门** - 手把手教你构建第一个小手机应用
- **核心架构** - 深入理解 C++ 核心层与 UI 层的设计
- **API 文档** - 完整的 API 参考和使用示例

## 本地开发

1. 安装依赖：
   ```bash
   npm install
   ```

2. 启动开发服务器：
   ```bash
   npm run docs:dev
   ```

3. 构建生产版本：
   ```bash
   npm run docs:build
   ```

## 部署设置

本项目配置了 GitHub Actions 自动部署到 GitHub Pages。首次部署需要进行以下设置：

1. 进入 GitHub 仓库的 **Settings** → **Pages**
2. 在 **Build and deployment** 部分，将 **Source** 选择为 **GitHub Actions**
3. 设置会自动生效，无需额外保存操作

> 💡 提示：如果遇到 `Error: HttpError: Resource not accessible by integration` 错误，通常是因为没有正确设置 Source。
